## Note pour l'administration fiscale

{: .short-version }

Ceci n'est pas un extrait fiscal officiel.
Ce document minimal ne contient que les codes-barres nécessaires pour importer les données bancaires par voie électronique.

Pour plus d'informations, voir la page "Note pour l'administration fiscale"

{: .long-version }


## But

Ceci n'est pas un extrait fiscal officiel. Il a été créé par le(s)
contribuable(s) à l'aide d'OpenSteuerauszug (https://github.com/vroonhof/opensteuerauszug) à partir de données
électroniques de l'institution financière. Il ne contient que des codes-barres afin
que les données bancaires puissent être importées dans le logiciel fiscal.

## Données

La responsabilité de l'exactitude et de l'exhaustivité incombe toujours au contribuable.

- Il/elle contrôle les positions, transactions, numéros de valeur et paiements d'intérêts.
- *Aucun calcul fiscal n'a été effectué.* Lorsque le format l'exige, les valeurs et totaux sont mis à zéro.
- Le calcul fiscal basé sur la Kursliste doit être repris par le logiciel fiscal et/ou le logiciel de taxation.

## Pourquoi un extrait fiscal ?

Malheureusement, il n'existe pas de moyen standardisé d'importer uniquement
les positions ou les transactions dans PrivateTax ou un logiciel similaire.

Les institutions financières étrangères ne fournissent pas d'extraits fiscaux électroniques.
L'auteur du logiciel OpenSteuerauszug estime qu'un transfert automatique des données
reste avantageux, d'autant plus que le logiciel fiscal est capable de corriger et/ou
de compléter les calculs fiscaux pertinents.
